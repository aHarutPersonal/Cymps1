Extract ACTIONABLE ACHIEVEMENTS and MILESTONES from the provided sources for the specified idol.

IDOL:
- idol_name: "{idol_name}"

INPUT SOURCES (chunked; each has source_id, source_url, chunk_index, text):
{sources_json_array}

GOAL:
Return a high-signal list of milestones that can be used for:
1) building an idol timeline,
2) comparing user vs idol at the same age,
3) deriving "replicable" behaviors for plan generation.

OUTPUT RULES (MANDATORY):
- Output MUST be valid JSON only (no markdown, no commentary).
- Output MUST match the schema exactly.
- Every candidate MUST include at least 1 evidence item.
- evidence.snippet MUST be an exact substring from the referenced chunk text (verbatim).
- Do NOT invent facts. If it's not in sources, omit it.

QUALITY FILTER: include only "achievement-grade" items
An item qualifies if it is one of:
- A concrete decision/action: founded, acquired, invested, launched, published, patented, led, hired, partnered, exited, built, restructured, expanded, won, introduced, created, pivoted.
- A measurable milestone: role appointment, deal completion, major publication, award/honor, key education/credential, notable product/company event, major philanthropic action.
- A clearly stated principle/strategy that is explicitly connected to actions (not generic traits).

EXCLUDE (NOT achievements):
- Birth/death events or basic biographical facts (born/died/grew up).
- Family/relationships unless explicitly tied to a concrete achievement (e.g., "co-founded with spouse" still needs the co-founding action).
- Pure personality descriptors (e.g., "known for being disciplined").
- Moves/relocations unless explicitly strategic and tied to a milestone.
- Rumors, speculation, "reportedly", or unverifiable claims.
- General claims without an action (e.g., "was influential").

ACTIONABILITY REQUIREMENT (CMPYS-specific):
Each candidate must be written so a user can "learn from it".
Therefore:
- title must be action-oriented ("Founded X", "Joined Y as Z", "Published A", "Pivoted strategy to B").
- description must contain:
  (a) WHAT they did (concrete action),
  (b) WHY it mattered (impact/trajectory),
  (c) a short "replicable angle" stated as a generalizable lesson, but WITHOUT inventing details.

Example structure (do NOT literally label it):
"Buffett formed Buffett Partnership Ltd. in 1956, creating a vehicle to apply value investing systematically. Replicable angle: start with a small, focused vehicle to compound results and iterate."

DATE RULES:
- If exact day is stated -> date="YYYY-MM-DD", date_precision="day"
- If only month+year -> date=null, date_precision="month"
- If only year -> date=null, date_precision="year"
- If no date -> date=null, date_precision="unknown"
Never guess missing dates or months/days.

IMPORTANT:
- age_at_event MUST remain null for all candidates (computed later).
- If the source explicitly states an age (e.g., "at 19"), you may mention it in description, but keep age_at_event null.

DE-DUPING (lightweight at extraction time):
- Avoid producing near-identical duplicates from the same source passage.
- If the same milestone appears multiple times, keep the clearest version and strongest evidence.

COUNT TARGET (signal over quantity — never invent to hit a number):
- Extract every genuinely achievement-grade, evidence-backed milestone you find, up to a cap of 25.
- When sources are rich this is typically 10–25 items.
- When sources are thin, return ONLY the items that are genuinely supported — even if that is just 2 or 3.
  Do NOT pad the list, restate the same milestone, or invent achievements to reach any minimum.
  An accurate short list is always better than a padded one.

CATEGORY RULES:
Pick exactly one:
- career: role appointments, leadership positions, company founding/management, partnerships, major career shifts
- learning: education, apprenticeships/mentors explicitly stated, key training/credential
- finance: investment actions, fund/partnership creation, acquisitions/sales, capital allocation milestones
- impact: philanthropy, institutions founded, policies influenced, large-scale societal contributions explicitly stated
- mindset: explicitly stated strategy/philosophy ONLY if linked to actions or decisions (not generic traits)
- other: notable items that don't fit above

IMPORTANCE HINT RULES (use conservatively):
- high: defining, trajectory-shaping milestone that appears central in sources
- medium: meaningful milestone with clear impact
- low: minor milestone or supporting detail (still must be real/actionable)

CONFIDENCE (0.0–1.0):
Set based on:
- evidence clarity (explicit vs implied),
- uniqueness (unambiguous vs ambiguous),
- date precision (day > month > year > unknown),
- source authority (Wikipedia lead/infobox vs vague mention).
Guidance:
- 0.90–1.00: explicit statement + clear milestone + strong evidence
- 0.70–0.89: clear but date coarse or phrasing indirect
- 0.40–0.69: partially supported or mildly ambiguous (use sparingly)
- <0.40: do not output

Return JSON EXACTLY matching this schema:

{
  "candidates": [
    {
      "title": "string (short, action-oriented, e.g., 'Founded Partnership', 'First Stock Purchase')",
      "description": "string (1-2 sentences: WHAT they did, WHY it mattered, plus a short replicable angle)",
      "date": "YYYY-MM-DD|null",
      "date_precision": "day|month|year|unknown",
      "age_at_event": null,
      "category": "career|learning|finance|impact|mindset|other",
      "importance_hint": "high|medium|low",
      "confidence": 0.0,
      "evidence": [
        {
          "source_id": "string",
          "chunk_index": 0,
          "source_url": "string",
          "snippet": "string (verbatim from source, max 240 chars)",
          "confidence": 0.0
        }
      ]
    }
  ]
}

EVIDENCE RULES:
- Provide 1–3 evidence items per candidate (prefer 1 strong, add more if corroborated).
- snippet must be <= 240 chars and copied verbatim from the referenced chunk.
- evidence.confidence should reflect how directly the snippet supports the candidate (often same as candidate confidence or slightly higher/lower).

FINAL CHECK BEFORE OUTPUT:
- Remove any item that is not a concrete action/milestone.
- Remove any item without a strong, verbatim evidence snippet.
- Ensure age_at_event is null for every candidate.
- Ensure JSON is valid and matches the schema exactly.
