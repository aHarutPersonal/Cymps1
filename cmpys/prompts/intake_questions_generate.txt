Generate an idol-specific intake questionnaire to personalize a 12-week plan.

INPUT:
- idol_name: "{idol_name}"
- idol_persona_json:
{idol_persona_json}
- idol_profile_json:
{idol_profile_json}
- milestones_json (optional):
{milestones_json}
- user_profile_json (optional):
{user_profile_json}
- target_age: {target_age}
- limit: {limit}

GOAL:
Ask the minimum number of high-signal questions (6–10) needed to:
- assess the user’s current skill level,
- capture real achievements/projects,
- identify constraints (time/money/tools),
- understand preferences (learning style),
- tailor to the idol’s style and frameworks.

IMPORTANT:
- Questions must be concrete, easy to answer, and suitable for mobile UI.
- If idol_persona_json indicates historical/non-modern, avoid modern corporate jargon and use the idol’s worldview metaphors lightly.
- Do NOT ask for sensitive personal data (passport, address, etc).

OUTPUT JSON ONLY:

{
  "questions": [
    {
      "id": "string",
      "title": "string",
      "prompt": "string",
      "type": "text|multiline|single_choice|multi_choice|scale",
      "required": true,
      "options": ["string"],
      "placeholder": "string|null",
      "validation": { "min_len": 0, "max_len": 0 },
      "category": "achievements|skills|constraints|goals|preferences|risk|resources",
      "mapping_hint": "string"
    }
  ]
}

RULES:
- 6 to 10 questions.
- Include at least:
  - 1 achievements/projects question (multiline)
  - 1 weekly time question (single_choice or text)
  - 1 skills self-rating question (scale)
  - 1 constraints question (multi_choice)
  - 1 goal question (text)
- Make mapping_hint explicit (e.g., "user_profile.weekly_hours", "user_profile.skills", "user_profile.achievements_raw").
- Questions should be specificly constructed for later comparision with that idol's milestones.
